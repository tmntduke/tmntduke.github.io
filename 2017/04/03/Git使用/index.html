git使用

创建版本库和提交

  • 创建

    使用cd进入项目目录中 之后 git init 创建代码仓库

  • 提交

    git add . 提交所有代码，add后加具体文件名为提交指定文件

    git add –A 当文档库中有文件删除 会扫描 并删除文件

    git add -u 查找更新

    git commit -m’说明’ 提交到版本库

删除和还原

当 add执行后：

删除 git rm (会删除文件) –cache (不会删文件 但会将文档库中的索引删除)

当commit执行后

删除 git reset 将还原commit操作

忽略

忽略上传文件:

在项目文件夹根路径下创建.gitignore文件，然后编辑文件，将要忽略的文件写入。

规则：

  • #任意字符

  • *代表忽略所有文件

  • file/ file文件夹下文件

  • /file 当前文件夹下file文件

  • /**/ 任意文件夹

查看文件修改情况

git status

commit 节点

HEAD 为提交节点 表示最近提交 ~为层级节点 ^为父节点
可简写 @

分支

branch –a 查看所有分支

git branch version1.0 创建分支

将代码转移到分支：git checkout version1.0

简便：
git checkout –b 直接创建新分支 并进入该分支

当文件添加到分支后 再切换分支 提交 此文件还会被提交到上一个分支

checkout 检出文件 （会还原文件）

删除分支：git branch –D version1.0

合并 冲突

合并

将两个分支的代码合并：

git checkout A

git merge B (将分支B合并到分支A)

之后将分支B删除

还原合并操作：git reset

git reset –hard HEAD^ 将文件一起还原

当分支是从master中生长出来 用以上方式合并
但是不会产生新的commit节点 同时不会留下合并的记录
只是将master引用到另一条分支上

还原

git revert 使文件还原到上一个commit节点 但不删掉节点 会产生一个新节点

保留合并记录：

git merge –no-ff 分支名称

产生新的commit节点 再master分支上

cherry-pick合并 将节点合并到文件中

产生新的commit节点 –n不产生

  1. 放弃 git cherry-pick abort
  2. 继续 修改冲突文件 执行git add. git cherry-pick continue

rebase将分支修改运用到另一个分支上

冲突解决和merge同

还原rebase git rebase –abort

已经执行完成 git reset HEAD（必需找到rebase之前的commit节点）

使用 git reflog HEAD 或分支名称

远程版本库

git init –bare Git 文档库文件夹名 （.git）

下载github上的代码：

git clone 远程地址 本地路径

本地文档库和远程文档库关联：

git remote add 远程文档库名称 远程地址

先进行add 和 commit

将代码上传到远程版本库：

git push 地址 分支

  1. push -u 与远程版本库分支取得联系，在配置文件中记录

  2. push –f 强制推送

将本地修改同步到远程版本库：git fetch 地址 分支

查看修改内容：git diff 地址/分支 之后用merge合并代码

获取最新代码并合并到分支: git pull 地址 分支

远程版本库新名称：

git remote rename newname

远程文档库新地址

git remote set-url newurl

删除远程某个分支:

git push 远程名称 –-delete 分支名称
